YOLO-Fine

This project analyzes New York City for-hire vehicle (FHV) trip data for May 2022 to uncover insights into trip patterns and compare the usage of different ride-sharing providers, primarily Uber and Lyft. The analysis involves extensive data cleaning, exploratory data analysis (EDA), and the creation of various visualizations to present the findings.

Project Summary

Project: Real-Time Pothole Detection with YOLOv11n

Project Overview

This project involved fine-tuning the state-of-the-art YOLOv11n (nano) object detection model to identify potholes in road images. The goal was to create a lightweight, efficient model capable of real-time detection, suitable for deployment on edge devices (like in-car systems or drones) to assist with infrastructure monitoring and driver alerts.

Objective

To develop a fast and accurate computer vision model that can detect and localize potholes from a single image, serving as a proof-of-concept for automated road maintenance and advanced driver-assistance systems (ADAS).

Dataset

The model was trained on a specialized Pothole Dataset sourced from Roboflow. An analysis of the dataset (labels.jpg) confirmed:

  • Single Class: The model was trained to detect one class: "pothole".
  • Dataset Size: The training set contained approximately 1,600 labeled instances.
  • Image Samples: The data consists of real-world road images with potholes of various shapes and sizes, as seen in the training batch sample.

Methodology & Training

  • Model: YOLOv11n, the smallest and fastest version in the YOLOv11 family, was chosen for its optimal balance of speed and accuracy, making it ideal for real-time applications.
  • Training: The model was fine-tuned for 50 epochs. The training progress (results.jpg) shows a consistent decrease in both training and validation loss (box, class, and DFL losses), indicating that the model learned to generalize well without significant overfitting.

Results & Evaluation

The model's performance was evaluated using standard object detection metrics on a held-out validation set. The results demonstrate a highly effective and well-balanced detector.

Key Performance Metrics

Based on the final training epoch and evaluation curves:

  • mAP@.5 (mean Average Precision at 50% IoU): 83.0%
  • mAP@.5-.95 (mean Average Precision, 50%-95% IoU): 41.0%
  • Precision: 86.4%
  • Recall: 70.1%

Precision-Recall (PR) Curve

The PR curve (BoxPR_curve.png) visualizes the trade-off between precision and recall for the "pothole" class. The large area under the curve (AUC), which corresponds to the mAP@.5 of 0.83, confirms the model's robustness. It maintains high precision even as recall increases, which is characteristic of a high-quality detector.

Confusion Matrix

The normalized confusion matrix (confusion_matrix_normalized.png) provides a clear breakdown of the model's detection accuracy on the validation set.

  • Correct Detections (True Positives): The model successfully identified 70% of all actual potholes.
  • Missed Detections (False Negatives): The model missed 30% of the potholes (classifying them as background).
  • Incorrect Detections (False Positives): A low "background FP" rate of 0.16 indicates that the model rarely made incorrect "pothole" predictions on background objects, which speaks to its high precision.

Conclusion

This project successfully demonstrates the power of the new YOLOv11n model for fine-tuning on a custom, real-world dataset. The resulting model achieves a strong balance of precision (86.4%) and recall (70.1%), culminating in a mAP@.5 of 83%.

This lightweight and efficient model is a strong candidate for real-world deployment in applications requiring rapid and reliable road hazard detection.
