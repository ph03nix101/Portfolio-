Credit Card Fraud Detection — Data Preprocessing & Model Evaluation

A complete data preprocessing and model evaluation notebook built on the Credit Card Fraud Detection dataset. The project focuses on preparing data, handling imbalance, and comparing multiple machine learning models to accurately detect fraudulent transactions.

Overview

This notebook demonstrates a full workflow for credit card fraud detection — from data loading and cleaning to feature preparation and model evaluation. The dataset used (mlg-ulb/creditcardfraud) contains 284,807 transactions with 31 features, including anonymized principal components (V1–V28), Time, Amount, and the target variable Class.

Key Steps

  • Data Loading: Retrieved dataset using KaggleHub and verified structure and datatypes with pandas.
  • Exploration: Reviewed data distribution, feature scales, and the extreme class imbalance between fraudulent and legitimate transactions.
  • Preprocessing: Applied transformations for model-readiness, including normalization, sampling, and preparation of training and test sets.
  • Model Training: Built and compared multiple algorithms — Logistic Regression, Random Forest, and XGBoost — for classification.
  • Evaluation: Computed metrics such as Precision, Recall, F1 Score, and ROC-AUC to assess fraud detection performance.

Results Summary

The notebook concludes with a model performance comparison:

Model Precision (Fraud=1) Recall (Fraud=1) F1 Score ROC-AUC
Random Forest 0.9481 0.7449 0.8343 0.9631
Logistic Regression 0.0563 0.9082 0.1060 0.9733
XGBoost 0.8737 0.8469 0.8601 0.9698

Among the models tested, XGBoost achieved the best overall balance between precision and recall, making it the most effective model for fraud detection in this study.

Tools & Libraries

Python, pandas, NumPy, XGBoost, scikit-learn, and KaggleHub for dataset management and modeling.

Conclusion

This project highlights the importance of preprocessing and model comparison when dealing with imbalanced datasets. By structuring a clear, reproducible pipeline, the notebook makes it easy to replicate results and extend the workflow to new fraud detection or anomaly detection problems.

from sklearn.metrics import precision_score, recall_score, f1_score, roc_auc_score
              def model_metrics(y_true, y_pred, y_proba):
                  precision = precision_score(y_true, y_pred)
                  recall = recall_score(y_true, y_pred)
                  f1 = f1_score(y_true, y_pred)
                  roc_auc = roc_auc_score(y_true, y_proba)
                  return precision, recall, f1, roc_auc
              precision_rf, recall_rf, f1_rf, roc_auc_rf = model_metrics(y_test, y_pred_rf, y_proba_rf)
              precision_lr, recall_lr, f1_lr, roc_auc_lr = model_metrics(y_test, y_pred_lr, y_proba_lr)
